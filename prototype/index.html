Static UI prototype JSON / local file storage in final build

Admin

Monitor accepted jobs and workload balance

Identify overload risk quickly and keep recruitment records visible.

Key Prototype Decisions

  • Role-based entry instead of full authentication
  • Explicit application statuses: Submitted, Under Review, Accepted, Rejected
  • Ranked preference when TA applies for a job
  • Workload summary with overload warning at 3 accepted jobs
  • Clear skill requirements in every job posting

Assessment Coverage

  • TA profile creation and CV upload
  • TA job browsing and application submission
  • MO posting creation and candidate review
  • Admin workload visibility
  • Validation and status feedback scenes

    Optional Enhancement

    AI features can be shown as optional decision support, not core workflow

    Coursework guidance does not require AI for full marks. The safest approach is to present AI as a modular enhancement that can be added after the core recruitment workflow is complete.

    Future enhancement No dependency for MVP
    TA-side feature Fit score Shows likely job fit before application
    MO-side feature Shortlist support Ranks applicants by skill fit and workload
    Admin-side feature Balancing advice Flags overload and suggests redistribution

    Recommended AI Features

    • Match a TA's declared skills against posting requirements
    • Highlight missing skills before TA submission
    • Give MO a workload-aware candidate summary
    • Suggest which accepted TAs may be near overload

    Safe Coursework Positioning

    • Keep AI outside MVP acceptance criteria
    • State that AI output is assistive, not authoritative
    • Do not require internet access for the base workflow
    • Prioritise core recruitment flow first, AI second
