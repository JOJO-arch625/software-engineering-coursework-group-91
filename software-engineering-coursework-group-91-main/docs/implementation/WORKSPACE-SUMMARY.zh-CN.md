# 阶段性中文总结报告

## 1. 项目业务流程

### 1.1 TA 业务流程
- TA 登录系统。
- 进入 TA 工作区后维护个人资料，包括姓名、学号、邮箱、电话、技能和可用时间。
- 上传本地 CV 文件，系统限制文件类型为 `PDF / DOC / DOCX`。
- 浏览当前开放的岗位列表，只显示可申请的岗位。
- 查看岗位详情，包括课程代码、技能要求、知识要求、工作量和截止时间。
- 选择申请优先级并提交申请。
- 在“我的申请”页面查看状态变化，包括 `Submitted`、`Under Review`、`Accepted`、`Rejected`。

### 1.2 MO 业务流程
- MO 登录系统。
- 进入 MO 工作区后查看自己负责的岗位概览。
- 创建、编辑、保存或关闭岗位。
- 查看每个岗位的申请人数和申请人详情。
- 审核申请人资料、技能、CV 路径和当前已录取工作量。
- 将申请状态更新为 `Under Review`、`Accepted` 或 `Rejected`。

### 1.3 Admin 业务流程
- Admin 登录系统。
- 进入 Admin 工作区后查看所有 TA 的已录取课程和数量。
- 识别达到或接近工作量阈值的 TA。
- 基于 workload 信息进行协调和风险提醒。
- Admin 当前不直接发布岗位，也不直接决定录取，只负责全局监控和分配平衡。

## 3. 技术栈

### 3.1 后端
- `Java 8`
- `Java Servlet`
- `JSP`
- `Tomcat Maven Plugin`

### 3.2 构建与运行
- `Maven`
- 本地运行使用 `tomcat7-maven-plugin`

### 3.3 数据存储
- 不使用数据库
- 使用本地 `JSON` 文件存储结构化数据
- 使用本地文件夹存储 CV 文件

### 3.4 前端
- JSP 服务端渲染页面
- 统一样式文件：`src/main/webapp/assets/styles/app.css`
- 少量原生 JavaScript 处理侧栏折叠和页面滚动按钮

## 4. 当前关键业务规则
- TA 最多申请 `3` 个岗位。
- TA 最多被录取 `3` 个岗位。
- 已关闭岗位不能继续申请。
- Rejected 状态不会被隐藏。
- Admin 根据已录取结果计算 workload。
- Search / Inbox / AI 目前不进入 MVP 核心验收。
