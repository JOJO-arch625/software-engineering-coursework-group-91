**JSP 与中英文文案绑定**

业务页面由 [BasePageServlet.java](c:/Users/Lidarou/Desktop/University/Year3_2/SoftwareEngineering/gp/software-engineering-coursework-group-91/src/main/java/com/group91/tars/servlet/BasePageServlet.java:27) 从 session 取出 `locale`，创建：

`I18n` 类通过基础名 `i18n.messages` 自动按 locale 加载 `messages_en.properties` 或 `messages_zh.properties`，位置在 [I18n.java](c:/Users/Lidarou/Desktop/University/Year3_2/SoftwareEngineering/gp/software-engineering-coursework-group-91/src/main/java/com/group91/tars/i18n/I18n.java:24)。

整体链路就是：

`JSP 切换按钮` → `?lang=en/zh` → `LocaleFilter 写 session.locale` → `BasePageServlet 创建 I18n` → `JSP 用 i18n.t(key)` → `messages_en/zh.properties 返回对应文案`。

是的，**实现位置不一样，但底层语言切换机制是同一套**。

`login.jsp` 比较特殊，因为它不是通过 `BasePageServlet.preparePage()` 注入 `i18n` 和 `lang` 的，所以它在页面里自己做了这几步：

[login.jsp](c:/Users/Lidarou/Desktop/University/Year3_2/SoftwareEngineering/gp/software-engineering-coursework-group-91/src/main/webapp/WEB-INF/jsp/login.jsp:5)

```jsp
java.util.Locale loginLocale = (java.util.Locale) request.getSession(true).getAttribute("locale");
if (loginLocale == null) { loginLocale = java.util.Locale.ENGLISH; }
I18n i18n = new I18n(loginLocale);
String lang = loginLocale.getLanguage();
```

也就是说，`login.jsp` 自己从 session 取 `locale`，自己 new `I18n`。

其他登录后的页面一般是：

`Servlet` → `BasePageServlet.preparePage()` → JSP

在 [BasePageServlet.java](c:/Users/Lidarou/Desktop/University/Year3_2/SoftwareEngineering/gp/software-engineering-coursework-group-91/src/main/java/com/group91/tars/servlet/BasePageServlet.java:27) 里统一做：

```java
Locale locale = (Locale) session.getAttribute(LocaleFilter.SESSION_LOCALE_KEY);
I18n i18n = new I18n(locale);
request.setAttribute("i18n", i18n);
request.setAttribute("lang", locale.getLanguage());
```

然后普通页面通过 `pageStart.jspf` 使用这个注入好的 `i18n` 和 `lang`。

所以可以理解为：

- `login.jsp`：自己初始化 `I18n`，自己生成语言切换按钮。
- `其他页面`：由 `BasePageServlet` 初始化 `I18n`，公共头部 `pageStart.jspf` 生成语言切换按钮。
- 两者最终都依赖同一个 `LocaleFilter` 处理 `?lang=en/zh`，并写入 session 的 `"locale"`。
- 两者最终也都用同一套资源文件：`messages_en.properties` 和 `messages_zh.properties`。

因此不是两套语言系统，只是 **登录页因为不走公共页面准备流程，所以把初始化逻辑写在 JSP 内部了**。
