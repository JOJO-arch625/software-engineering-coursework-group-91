### 2. LoginServlet.doGet() 处理

[LoginServlet.java:14-24](../../src/main/java/com/group91/tars/servlet/LoginServlet.java#L14-L24)

```java
protected void doGet(HttpServletRequest request, HttpServletResponse response)
    throws ServletException, IOException {
    // 1. 检查 Session 中是否已有登录用户
    UserAccount currentUser = getCurrentUser(request);
    if (currentUser != null) {
        // 已登录 → 直接跳转到对应角色的首页
        redirect(request, response, service.getHomePathForRole(currentUser.getRole()));
        return;
    }
    // 2. 未登录 → 取出 Flash 消息（如果有的话）并显示 login.jsp
    request.setAttribute("flash", FlashScope.consume(request));
    forward(request, response, "/WEB-INF/jsp/login.jsp");
}
```

**逻辑分支：**
- **已登录用户**：跳转走，不显示登录页。TA → `/ta/dashboard`，MO → `/mo/dashboard`，ADMIN → `/admin/workload`
- **未登录用户**：消费上次可能残留的 Flash 消息，渲染 login.jsp

## 第二阶段：用户提交登录表单 (POST /login)

### 4. LoginServlet.doPost() 验证密码

[LoginServlet.java:26-39](../../src/main/java/com/group91/tars/servlet/LoginServlet.java#L26-L39)

```java
protected void doPost(HttpServletRequest request, HttpServletResponse response)
    throws IOException {
    // ① 调用 service.authenticate() 验证用户名密码
    UserAccount account = service.authenticate(
        request.getParameter("username"),
        request.getParameter("password")
    );

    if (account == null) {
        // ② 验证失败 → 设置错误 Flash 消息，重定向回 /login
        flashI18n(request, "error", "flash.login.invalid");
        redirect(request, response, "/login");
        return;
    }

    // ③ 验证成功 → 将 UserAccount 存入 Session
    request.getSession(true).setAttribute(SESSION_USER, account);  // SESSION_USER = "currentUser"

    // ④ 设置欢迎 Flash 消息
    flashI18n(request, "success", "flash.login.welcome", account.getDisplayName());

    // ⑤ 跳转到角色对应首页
    redirect(request, response, service.getHomePathForRole(account.getRole()));
}
```

**关键操作：**

| 步骤 | 失败分支 | 成功分支 |
|---|---|---|
| 验证 | `authenticate()` 返回 `null` | 返回 `UserAccount` 对象 |
| 消息 | Flash 错误消息 → "Invalid username or password" | Flash 成功消息 → "Welcome back, xxx" |
| 跳转 | `redirect` → `/login`（回到登录页） | `redirect` → 角色首页 |
| Session | 不写入 | `session.setAttribute("currentUser", account)` |

### 5. TarsService.authenticate() 核心验证

[TarsService.java:50-62](../../src/main/java/com/group91/tars/service/TarsService.java#L50-L62)

```java
public UserAccount authenticate(String username, String password) {
    // 空值检查
    if (isBlank(username) || isBlank(password)) {
        return null;
    }

    // 遍历所有账户，逐个比对
    for (UserAccount account : store.loadAccounts()) {
        if (username.trim().equalsIgnoreCase(account.getUsername())  // 用户名不区分大小写
            && password.equals(account.getPassword())) {               // 密码区分大小写，明文比对
            return account;
        }
    }
    return null;  // 没匹配到 → 返回 null
}
```

**验证规则：**
- 用户名：忽略大小写 + 去首尾空格（`equalsIgnoreCase`）
- 密码：严格区分大小写，**明文比对**（无加密/哈希）
- 任一字段为空 → 直接返回 `null`

`readList()` 方法（第113-123行）：
```java
private <T> List<T> readList(Path path, Type type) {
    if (!Files.exists(path)) {
        return new ArrayList<T>();     // 文件不存在 → 空列表
    }
    try (Reader reader = Files.newBufferedReader(path, StandardCharsets.UTF_8)) {
        List<T> data = gson.fromJson(reader, type);
        return data == null ? new ArrayList<T>() : data;
    }
}
```

## Flash 消息机制

登录过程中的错误/成功提示依赖 `FlashScope`：

[FlashScope.java](../../src/main/java/com/group91/tars/web/FlashScope.java)

```
POST /login 失败
  → flashI18n(request, "error", "flash.login.invalid")
    → FlashScope.put(request, "error", i18n.t("flash.login.invalid"))
      → session.setAttribute("flashMessage", new FlashMessage("error", "Invalid username..."))
  → redirect → /login

GET /login (新请求)
  → FlashScope.consume(request)
    → FlashMessage msg = session.getAttribute("flashMessage")
    → session.removeAttribute("flashMessage")   // 取出后立即删除
    → 返回 msg 给 JSP 渲染
```

**设计要点**：Flash 消息存入 Session → 重定向后取出 → 立即删除。这样刷新页面不会重复显示。

## 安全分析

| 问题 | 说明 |
|---|---|
| **密码明文存储** | `accounts.json` 中密码直接以明文 `"TaDemo123"` 保存，无哈希 |
| **密码明文比对** | `authenticate()` 中用 `password.equals(account.getPassword())` 直接比较 |
| **无防暴力破解** | 没有登录失败次数限制、验证码、账户锁定等机制 |
| **Session 固定** | 没有在登录后调用 `session.invalidate()` + 新建 session，存在 Session Fixation 风险 |
| **无 HTTPS 强制** | 表单提交无加密传输保障 |
| **用户名枚举** | 错误消息统一为 "Invalid username or password"，未区分用户不存在 vs 密码错误（这一点是好的） |

> 这些是为教学演示目的的有意简化，不应直接用于生产环境。
