## Page Goal

- My page will teach colour encoding concepts through interactive mini-games.
- The main aim is to make colour models easier to understand through direct user interaction, not only text explanation.

## Planned Page Structure

1. Header with navigation, language switch, font-size control, and sound toggle
2. Hero section introducing the game page
3. Score and status strip
4. Game selector
5. Mini-game 1: Model Match
6. Mini-game 2: Hue Lab
7. Safety and cross-page support panels

## Core Game 1: Model Match

### Goal

- Help users match colour models to real media or design scenarios.

### Rules

- The user sees one scenario.
- The user chooses one answer from `RGB`, `CMYK`, `HSV`, and `YCbCr`.
- The page gives immediate feedback and explains the correct choice.

### Input

- Click one option button.

### Feedback

- Correct answer:
  - highlight the correct option
  - increase score
  - increase streak
  - show explanation
- Incorrect answer:
  - show incorrect choice
  - reveal correct answer
  - reset streak
  - explain why the correct model is better

## Core Game 2: Hue Lab

### Goal

- Help users understand hue, saturation, and lightness by adjusting colour directly.

### Rules

- The user sees a target colour.
- The user adjusts `Hue`, `Saturation`, and `Lightness` sliders.
- The user presses a check button.
- The system returns a match percentage and feedback.

### Input

- Slider movement
- Check button
- Reset button

### Feedback

- High match: praise + higher score
- Medium match: encouragement + moderate score
- Low match: retry guidance + streak reset

## Accessibility and Design Considerations

- English and Chinese support
- Adjustable font size
- Responsive layout for desktop and mobile
- Clear and safe interaction
- No login and no personal data collection
